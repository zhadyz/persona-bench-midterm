Key Principle

Anticipated Questions

Q: Why 70 per persona instead of 100?
We prioritized coverage of all preference categories + dynamic events first. Expanding to 100 means adding 30 more Type 4 reasoning cases per persona — straightforward once the approach is validated.
Q: Why is accuracy 60% instead of 40-50%?
Because the app logs embed themes at a high level (the LLM can always find something), but lack the concrete details needed for CORRECT scores. With more specific facts in the logs, some PARTIALs become CORRECT and some become INCORRECT, bringing accuracy into the target range.
Q: How do you know the test cases are fair?
Every ground truth answer is sourced from the verified extracted profile (Step 2b). Every question is designed to be answerable from app logs alone — no seed data or profile knowledge required. The 4-type distribution ensures a difficulty gradient from trivial retrieval to deep inference.
Q: What if Anmol's logs change? Do you have to redo test cases?
No. The test cases test the persona, not the logs. The ground truth doesn't change. What changes is accuracy — better logs = more facts findable = accuracy adjusts. We just re-run the benchmark.
Q: How do we handle the cross-session contamination problem (Gemini issue)?
Each benchmark run uses a fresh session (new chat, or fresh API call). No prior context. This is automatic when using API calls. For manual testing: new chat window per persona.
Q: How do we scale to 100 personas?
Automated pipeline: Python script makes API calls (one per test case per persona). Each call gets fresh context. Can test across Claude, GPT-4o, Gemini simultaneously. ~3,000-5,000 API calls total, or use an agent orchestration system.
Q: Are the dynamic events from Anmol good enough?
The 6 events per persona are well-designed and plausible. The issue is execution: events are referenced but don't always progress with new facts over time. Early-stage sessions (setup, coordination) are good; late-stage (debrief) sessions recycle content. Anmol is aware and working on it.
Q: What's the relationship between preferences and dynamic events in the test cases?
Preferences are static facts (e.g., "Julio shops at Rochester Public Market"). Dynamic events change something (e.g., "Karl's knee consult shifted Julio's schedule"). Type 3 test cases specifically test whether the LLM can detect these changes. Type 4 cases require reasoning across both static and dynamic information.
Q: Why did you challenge Anmol's logs?
Per professor's instruction: evaluate app log quality before generating test cases. Found: zero dollar amounts across all 5 files, contact distribution heavily skewed (one contact gets 50-72% of sessions), and conversations are abstract rather than behaviorally specific. Anmol has already fixed duplication (was 22.5%, now 0%) and is iterating on the remaining issues.
Q: How does this connect to the Stanford paper (Park et al.)?
Park et al. achieved ~85% accuracy with rich context, normalized against 81% human self-agreement. Our 40-50% target is for a harder task: inferring persona from indirect behavioral evidence in app logs, not from direct profile data. The 4 test case types mirror the paper's finding that context beats demographics — our Type 4 reasoning cases specifically test contextual inference.

Type 1 — Simple Fact-Check (Easy)
Q: Julio has a recurring social commitment on Thursday evenings. What is it?
Ground Truth: Biweekly D&D campaign at Marcus Chen's house. Julio is the Dungeon Master.
Evidence: Calendar entries for "Campaign Night at Marcus's"
Type 2 — Cross-Log Fact-Check (Medium)
Q: A family member had a medical appointment this month. Who, what for, and how did Julio help?
Ground Truth: Father Karl — orthopedic consult for knee. Julio handled rides, groceries, compiled surgery questions. Coordinated with sister Greta.
Evidence: Calendar "Karl Orthopedic Consult" + "Pharmacy + Groceries" + messenger sessions with Greta and Eileen
Type 3 — Dynamic Tracking (Medium-Hard)
Q: Julio's health routine shifted mid-month. What changed?
Ground Truth: Before: pushing through, overextended nights. After: lighter routine — morning lamp, shorter canal walks, acupuncture as maintenance, deliberate meals.
Evidence: Pre-D4 calendar intensity vs. post-D4 "Canal Walk - Short Loop", "Acupuncture Reset", "Quiet Dinner + Lamp Reset"
Type 4 — Reasoning (Hard)
Q: Construct a full psychological profile of Julio from the logs alone.
Ground Truth: High Openness, variable Conscientiousness, moderate Extraversion, moderate-high Agreeableness, low Emotional Stability. Strengths: creative versatility, community-building. Vulnerabilities: financial precarity, perfectionism, intensity-crash cycles.
Evidence: Full synthesis across all themes, events, calendar patterns, and conversations
